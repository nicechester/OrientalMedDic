# 📱 온디바이스 한의·중의 통합 한자 사전 앱 개발 계획서

## 1. 제품 정의 및 핵심 가치 (Core Value)

* **목표:** 단순 낱글자 자전을 넘어, 의서(醫書)의 복잡한 한문 문장이나 처방전을 카메라로 찍으면 문맥을 파악해 정확한 한의학 용어로 해설해 주는 온디바이스 앱.
* **차별화:** 일반 번역기(구글, 파파고)가 잡지 못하는 동양의학 특유의 다의어 및 도메인 지식(예: '氣'나 '表裏'의 의학적 맥락)을 로컬 환경에서 완벽히 소화.

---

## 2. 하이브리드 시스템 아키텍처

500MB 체급의 모델을 유지하기 위해, 단순 매핑이 가능한 지식은 DB로 빼고 LLM은 '맥락 해석'에만 집중시킵니다.

```
[카메라 촬영 / 텍스트 입력]
         ⬇️
1. 온디바이스 OCR ➡️ 한자 텍스트 추출 (간체/번체 동시 대응)
         ⬇️
2. 라우팅 및 텍스트 전처리 (Routing)
   ├── Case A: 단일 한자어 검색 ➡️ [SQLite 사전 DB] ➡️ 즉시 출력 (음/뜻, 본초/방제 기초 정보)
   └── Case B: 의서 문장 / 처방 구절 ➡️ [온디바이스 Qwen LLM] ➡️ 문맥 번역 및 한의학적 해설 출력

```

## 3. 컴포넌트별 상세 개발 계획

### ① 이미지 및 OCR 모듈 (간체/번체 통합)

중국 의서는 간체자, 국내 고의서는 번체자(우리가 아는 한자)나 이체자로 되어 있습니다. 이 둘을 모두 매끄럽게 텍스트로 전환해야 합니다.

* **기술 스택:** iOS `Vision Framework` / Android `Google ML Kit` (중국어 및 한국어 팩 필수 포함).
* **간·번체 정규화 (Normalization):** OCR 직후, 내부 매핑 테이블을 거쳐 간체자를 국내 한의학 표준 한자(번체)로 1차 정규화하여 DB 검색 및 LLM 입력의 정확도를 높입니다.

### ② 로컬 구조화 데이터베이스 (SQLite, 약 10~15MB)

LLM의 리소스를 아끼기 위해 명사형 데이터는 전부 DB화합니다.

* **한방 본초(약재) 사전:** 약 500여 종의 주요 한약재 이름, 기미(氣味), 귀경(歸經), 효능.
* **표준 방제(처방) 사전:** 《방약합편》, 《동의보감》 등에 수록된 주요 처방 구성 및 주치 증상.
* **상용 한자 사전:** 기본 자전 데이터.

### ③ 온디바이스 LLM 최적화 (약 350MB ~ 1GB)

의학 문맥을 짚어내기 위해 **Qwen 2.5/3.5 계열**을 메인으로 가져갑니다.

* **선택 옵션 1 (극초경량):** `Qwen 2.5-0.5B-Instruct` ➡️ 4-bit 양자화 시 **약 390MB**. 속도가 매우 빠르고 용량 부담이 전혀 없음.
* **선택 옵션 2 (성능 밸런스형):** `Qwen 2.5-1.5B-Instruct` ➡️ 4-bit 양자화 시 **약 980MB**. 문장 분석력이 0.5B보다 확연히 뛰어남. 성능을 고려하면 1.5B 체급을 4-bit(Q4_K_M)로 깎아 1GB 이하로 맞추는 방향을 추천합니다.

## 4. 데이터 수집 및 파인튜닝 (SFT) 전략

이 앱의 핵심은 "중의/한의 의학 문헌 데이터로 LLM을 학습시키는 것"입니다.

### ① 학습 데이터 소스 (Data Sources)

* **AI Hub 공공 데이터:** `고전 문헌 번역 데이터셋` 및 `한문-한국어 병렬 코퍼스`. (한문 특유의 구조와 문법 체계를 모델에게 각인시키는 용도)
* **한국한의학연구원(KIOM) / 전통의학 정보포털:** 《동의보감》, 《경악전서》, 《상한론》 등 주요 핵심 의서의 원문-국역 병렬 데이터셋 추출.
* **중의학 오픈소스 (GitHub):** 중국 내에서 공개된 중의학 기초이론 및 진단학 Q&A 데이터셋을 확보한 후, 의학 용어를 한국 표준 한의학 용어로 번역 가공.

### ② SFT 프롬프트 데이터 포맷 (JSON)

단순 직역이 아니라 **의학적 맥락**을 짚어주도록 데이터 구조를 설계합니다.

```json
[
  {
    "instruction": "다음 동양의학 문장을 한의학적 맥락을 살려 한국어로 번역하고 해설해줘.",
    "input": "治𣩻(𣩻) 補氣血 淸濕熱",
    "output": "[독음]\n치위 보기혈 청습열\n\n[한의학적 해석]\n위증(𣩻症: 사지가 늘어지고 힘이 없어 쓰지 못하는 병증)을 치료할 때는 기와 혈을 보하고, 습열(濕熱: 습기와 열기)을 맑게 제거해야 유효하다는 뜻입니다. 즉, 몸의 면역과 영양을 채우면서도 염증성 노폐물을 끄는 치료 원칙을 제시하고 있습니다."
  }
]

```

### ③ 학습 및 변환 파이프라인

1. **M3 Max 64GB 환경**에서 `Unsloth` 또는 `MLX` 라이브러리를 사용해 LoRA 파인튜닝 진행.
2. 의서 특유의 특수 한자나 이체자를 깨뜨리지 않도록 Qwen의 **Tokenizer(Vocab 사이즈 151,643)** 설정을 그대로 유지하며 학습.
3. 학습 완료 후 `llama.cpp`를 통해 모바일 NPU 가속에 최적화된 **GGUF format**으로 4-bit 양자화 변환.

## 5. UI/UX 및 클라이언트 구현 개발 스택

* **iOS 스택:** SwiftUI + Vision Framework + **MLX-Swift** (Apple Neural Engine을 직접 타격하여 실시간 인터랙티브 추론 구현).
* **핵심 UX:**
* **카메라 뷰 파인더 라이브 스캔:** 책에 카메라를 대면 혈자리, 약재, 의서 문장 영역이 박스로 표시됨.
* **단어 탭 팝업:** 문장 중 특정 단어를 누르면 SQLite DB에서 가져온 낱글자 뜻이 0.1초 만에 팝업으로 노출.
* **통합 풀이 시트:** 하단 시트를 위로 올리면 온디바이스 LLM이 분석한 전체 문장의 의학적 해설이 매끄럽게 출력.

## 6. 마일스톤 및 즉시 시작 가능한 액션 플랜

1. **의서 병렬 데이터 수집 (1주):** AI Hub 및 한의학 오픈 포털에서 고전 의서 한자-한글 데이터 2만 건 이상 확보.
2. **M3 Max에서 베이스 모델 추론 테스트 (3일):** 양자화되지 않은 오리지널 `Qwen 2.5-1.5B-Instruct` 모델에 한방 의서 구절을 몇 개 던져보고, 제로샷(Zero-shot) 상태에서의 번역 능력을 먼저 측정. (이를 통해 파인튜닝 데이터의 밀도를 결정)
3. **LoRA 학습 및 양자화 (1주):** 수집된 데이터를 포맷팅하여 로컬 학습 진행 후 모바일용 파일로 변환.
